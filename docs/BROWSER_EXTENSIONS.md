# 浏览器扩展对开发体验的影响

## 现象

部分浏览器扩展（如 Zotero、广告拦截器、翻译插件等）会在页面中注入脚本或修改 DOM，导致：

- `Could not establish connection. Receiving end does not exist` 错误
- 控制台错误不断刷新
- Hydration 报错（React 服务端与客户端 DOM 不一致）
- 页面卡顿、内存占用升高

## 是否影响性能？

**会有影响**。扩展反复尝试连接、注入失败、重试，会：

- 占用 CPU
- 增加内存
- 拖慢页面渲染
- 产生大量控制台噪音，干扰调试

## 如何避免

### 方案 1：使用无痕/隐私模式（推荐）

Chrome / Edge 无痕模式下，扩展默认不运行（或可单独配置）：

1. 打开无痕窗口：`Ctrl+Shift+N`（Windows）或 `Cmd+Shift+N`（Mac）
2. 在无痕窗口中访问 `http://localhost:3000` 进行开发
3. 如需在无痕中使用少量扩展，可在 `chrome://extensions` 中勾选「在无痕模式下启用」

### 方案 2：单独的开发用浏览器配置

创建一个干净配置，用于开发：

- **Chrome**：右上角头像 → 添加 → 新建配置，命名为「开发」
- **Firefox**：使用「Firefox 开发者版」或新建配置文件
- 在「开发」配置中只安装必要的扩展（如 React DevTools），不安装 Zotero、广告拦截器等

### 方案 3：在扩展中排除本地站点

部分扩展支持排除列表：

- **Zotero Connector**：Zotero 设置 → Connector → 排除站点
- 添加 `localhost`、`127.0.0.1`、`47.96.75.69` 等你的开发/部署域名

### 方案 4：开发时临时禁用扩展

- 打开 `chrome://extensions`
- 临时关闭 Zotero、广告拦截器、翻译插件等
- 开发完成后再开启

## 总结

扩展的行为在浏览器层面，应用本身无法阻止。建议用**无痕模式**或**单独开发配置**进行开发，减少干扰。
