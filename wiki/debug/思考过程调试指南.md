# 思考过程 (think) 调试指南

## 1. WebSocket HMR 错误说明

```
WebSocket connection to 'wss://career.soulhappylab.com/_next/webpack-hmr' failed
```

**是否合理？** 是的，在以下场景会常出现：

- **开发模式** 下通过 `career.soulhappylab.com` 访问，但 dev server 在本地或其他地址
- 反向代理 / Nginx 未正确转发 WebSocket 到 dev server
- 使用生产构建时，浏览器仍尝试连接开发环境的 HMR

**影响**：只会影响热更新，不影响主业务逻辑，可暂时忽略。

## 2. completion_check 日志说明

```
[completion_check] 同步检测 user_count=3 conclusion_shown_at=None explicit=False should_check=False
```

- `should_check=False` 表示**不会**走结论卡路径（`check_dimension_complete`）
- 请求会继续走到**主流式对话**：`llm.chat_stream()` → 产生 think 事件

因此这条日志本身是**正常的**，且说明当前应走的是带思考过程的主对话流。

## 4. 建议调试顺序

1. **确认模型**  
   在 `simple_chat.py` 第 1596 行打断点，查看 `llm.model` 是否为 `deepseek-reasoner`。

2. **确认推理分支**  
   在 `openai_provider.py` 第 144 行打断点，若 `_is_reasoning_model()` 为 False，说明没有进入推理逻辑。

3. **确认 API 返回**  
   在 `openai_provider.py` 第 189 行打断点，观察 `rc` 和 `cc`：
   - `rc` 有值 → 有 `reasoning_content`，应能 yield `think_start` / `think_end`
   - 始终为空 → 可能是 API 或 SDK 未返回 `reasoning_content`

4. **确认 SSE 事件**  
   在浏览器 Network 面板查看 `/api/v1/simple-chat/message/stream` 的 EventStream，搜索 `think_start`、`think_end` 是否出现。
