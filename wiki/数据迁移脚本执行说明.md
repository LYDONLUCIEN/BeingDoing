# 数据迁移脚本执行说明

> 本文档记录所有数据迁移脚本的用途、执行方式和注意事项。
> 执行前请务必备份数据目录。

---

## 迁移脚本列表

### 1. `migrate_purpose_value_to_values.py`

**创建日期**: 2026-05-22
**关联 Issue**: #5 (R5-21-03 / R5-21-04)
**用途**: 将 purpose 结论卡中 `experience_value_rows` 的旧格式 `{value: "xxx"}` 转为新格式 `{values: ["xxx"]}`。

**变更内容**:
- 扫描所有 report 的 purpose thread JSON 文件
- 找到 `role: "conclusion_card"` 消息中的 `experience_value_rows`
- 将 `{experience: "...", value: "成长"}` 转为 `{experience: "...", values: ["成长"]}`
- 已是新格式（含 `values` 字段）的行不做改动

**执行命令**:

```bash
# 1. 先预览，确认影响范围（不实际修改文件）
cd src/backend && python scripts/migrate_purpose_value_to_values.py --dry-run

# 2. 确认无误后执行迁移
cd src/backend && python scripts/migrate_purpose_value_to_values.py
```

**前置条件**:
- 确保已备份 `data/simple/reports/` 目录
- 确认当前无用户正在进行 purpose 阶段对话

**回滚方式**: 从备份恢复 `data/simple/reports/` 目录。

## 通用注意事项

1. **执行前备份**: 所有迁移脚本执行前，请备份 `data/simple/` 目录
2. **dry-run 先行**: 先用 `--dry-run` 预览，确认影响范围
3. **停服执行**: 建议在低峰期执行，避免与用户对话冲突
4. **执行后验证**: 迁移后抽样检查数据完整性
