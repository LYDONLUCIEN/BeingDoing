**开场祝贺**（引号统一为直角引号）：

```diff
+ "恭喜你即将完成整个职业探索旅程！本轮我们将一起发现你的工作使命——你内心深处最希望为他人提供的价值。"
```

**价值观匹配规则**：
```diff
- 向用户确认："在这段经历中，你帮助他人获得的成果/感受，是不是对应着[某个价值观]？你觉得这个匹配准确吗？"
- 如果用户认可，则记录该经历与对应的价值观。如果用户不认可，继续引导用户思考更匹配的价值观，直到用户确认。
+ 向用户确认："在这段经历中，你帮助他人获得的成果/感受，对应着哪些价值观？你觉得这个匹配准确吗？"
+ 一段经历可以匹配多个价值观，不需要区分主次——用户认为哪个价值观在其中得到体现，就记录下来即可。
+ 如果用户不认可，继续引导用户思考更匹配的价值观，直到用户确认。
```

**归纳总结 — 禁止统计**：
```diff
- 4. **统计与总结**：
-     - 完成10段经历的分析后，统计每个价值观关键词出现的次数。
-     - 根据统计结果，为用户整理一份使命总结，内容包括：
-       - **（1）经历-价值观对应表格**
-       - **核心价值概括**：用一句话概括你最希望传递的核心价值观
-       - **命名约束**使命相关表述中的核心词条须为单一概念，不得使用「/、或、以及、&、|」并列多个候选
+ 4. **归纳与总结**（内部推理与用户展示分离）：
+     - **内部推理（不输出给用户）**：完成10段经历的分析后，在推理层面对各价值观出现的频率与分布进行统计
+     - **对用户展示**：向用户呈现使命总结时，采用归纳式表达，**禁止向用户展示任何统计次数、频次或计数信息**
+       - **（1）主题归纳**：用自然语言归纳共同主题和模式
+       - **使命概括**：用一到两句话概括（可以是一个，也可以是多个并列）
```

```diff
- - 【命名约束】使命相关表述中的核心词条须为单一概念，不得使用「/、或、以及、&、|」并列多个候选；近义词须与用户确认取舍后再定稿。
```

## 改动总结

| 改动类别 | 核心变化 |
|---------|---------|
| **冷启动容错** | 5 个阶段开场均增加"不确定也没关系"引导；values/strengths 明确告知可说"不知道" |
| **使命阶段统计禁止** | 使命总结从"统计+计数"改为"归纳式表达"，禁止向用户展示出现次数/频次 |
| **使命允许多价值观** | 一段经历可匹配多个价值观、使命可包含多个价值观、不再要求"单一概念"命名约束 |
| **下拉文案统一** | "其他"→"自定义"、"暂未选定"→"无"、"待定"→"无"，含历史值兼容映射 |
| **步骤 2 匹配标准** | 新增明确判断标准说明（互相增强=匹配、难以协同=不匹配），移除"匹配原因"列 |
| **深入聊天差异化** | 4 个步骤各自拥有独立 system prompt 片段，替代旧版统一模板 |
| **闸门防重复弹出** | 引入 `neg_gate_triggered_steps` 标记，重新填写时自动重置 |
| **价值观关键词快照** | Step 4 首次提交时保存快照，后续读取优先用快照保证下拉与对话一致 |
| **登录跳转统一** | 所有 auth 入口默认 redirectTo=`/explore/intro`，移除"回到上次页面"逻辑 |
