### O-03：激活码归属安全 + 审计日志

**验收标准（来自 4.28 清单）：**
- 正常激活码可正常使用，归属不变
- 激活码归属变更操作有审计日志（code、old_owner、new_owner、时间）
- 不会出现非预期的归属迁移

**测试用例：**

```
test_o03_is_owner_returns_true_for_matching_user
test_o03_is_owner_returns_false_for_different_user
test_o03_is_owner_returns_false_when_no_owner
test_o03_claim_owner_sets_user_fields
test_o03_claim_owner_triggers_audit_log
test_o03_claim_owner_audit_log_contains_required_fields  # code, old_owner, new_owner, time
test_o03_get_activation_returns_none_for_nonexistent
test_o03_unauthorized_claim_does_not_overwrite
```

**Mock 策略：** 用 `tmp_path` 作为激活码数据目录，直接走真实文件读写（激活码模块基于 JSON 文件，不需要数据库）。审计日志同样写入 `tmp_path`。

**测试文件：** `backend/test_activation_security.py`

### S-06：回看（只读）vs 重填路径

**验收标准：**

- 重新填写：显式重置当前阶段快照，允许重新填写

**测试用例：**

```
test_s06_lock_previous_step_when_entering_next
test_s06_lock_previous_step_does_not_lock_current
test_s06_lock_step1_then_step2_step1_locked
test_s06_first_step_never_locked
test_s06_reenter_same_step_not_locked
```

**Mock 策略：** `lock_previous_step_when_entering` 写入 JSON 文件。用 `tmp_path` 作为 `reports_root`。

**测试文件：** `backend/test_journey_resume.py`（与 S-05 共享）

## 第三批：E2E / 前端测试（3 个 Task ID）

### O-02：登录后统一跳"开始探索"

**验收标准：**
- 登录后统一跳转到"开始探索"页
- 清缓存后重新登录，跳转目标仍是"开始探索"
- 无残留的"回到上次页面"逻辑

**实现方式：** Playwright E2E
- 需要前端 dev server 运行中
- 需要测试账号或 mock auth
- 文件：`e2e/test_login_redirect.spec.ts`

**前置条件：** `npm install -D @playwright/test` + `npx playwright install chromium`

### O-06：跨设备/清缓存恢复

**验收标准：**
- 删除对话后刷新，该对话不再出现
- 跨设备登录后 Journey 状态一致
- 清缓存后不丢失已保存数据

**实现方式：** Playwright E2E
- 需要 backend + frontend 运行中
