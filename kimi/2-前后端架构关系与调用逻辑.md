## 二、前端架构详解

### 2.1 技术栈

| 层级 | 技术 | 用途 |
|-----|------|------|
| 框架 | Next.js 14 (App Router) | React服务端渲染框架 |
| 语言 | TypeScript | 类型安全 |
| 样式 | Tailwind CSS | 原子化CSS |
| 状态 | Zustand | 轻量级状态管理 |
| 表单 | React Hook Form + Zod | 表单验证 |
| 动画 | Framer Motion | 动画效果 |
| HTTP | Fetch API | API请求 |
| **i18n** | **自定义** | **中英文切换（新增）** |

### 2.2 目录职责

```

              │验证激活码│   │保存基本信息│  │ values对话   │
```

### 4.2 核心功能调用链

#### 4.2.1 简单模式对话流程（当前主要使用）

```
前端: app/explore/chat/[phase]/page.tsx
  │
  │ POST /api/v1/simple-chat/message/stream
  │ {activation_code, message, phase, thread_id}
  ▼
后端: api/v1/simple_chat.py::simple_chat_stream()
  │
  ├─→ 1. 验证激活码 (SimpleActivationManager)
  ├─→ 2. 加载历史消息 (ConversationFileManager)
  │      └─→ 文件级并发锁（按report_id+category）
  ├─→ 3. 构建 system_prompt (_build_system_prompt)
  │      ├─ 加载锚点摘要（若存在）
  │      ├─ 截断历史（最近20轮）
  │      ├─ 加载题目库 (KnowledgeLoader)
  │      ├─ 加载基本信息 (load_basic_info)
  │      └─ 加载上一轮结果 (load_prior_context)
  ├─→ 4. 调用 LLM (llm.chat_stream)
  └─→ 5. 流式返回并保存消息
  │
  ▼
数据: data/simple/{session_id}/
      data/simple/reports/{report_id}/
```

#### 4.2.2 完整模式对话流程（LangGraph，当前备用）

```
前端: app/dashboard/guide/page.tsx (或旧版探索页)
  │
  │ POST /api/v1/chat/messages/stream
  │ {session_id, message, current_step}
  ▼
后端: api/v1/chat.py::send_message_stream()
  │
  ├─→ 1. 创建 Agent Graph (create_agent_graph)
  │      ├─ reasoning_node (推理)
  │      ├─ action_node (行动)
  │      ├─ observation_node (观察)
  │      └─ user_agent_node (用户代理)
  ├─→ 2. 创建初始状态 (create_initial_state)
  ├─→ 3. 运行 Graph (graph.astream)
  │      ├─ reasoning: 分析输入、决定行动
  │      ├─ action: 执行工具（搜索/引导）
  │      ├─ observation: 观察结果
  │      └─ 循环直到结束
  └─→ 4. 返回结果并保存
  │
  ▼
数据: data/conversations/{session_id}/
```

#### 4.2.3 答题卡生成流程

```
用户对话 (多轮)
  │
  ▼
should_show_answer_card() 判断充分性
  │
  ├─ 条件1: 对话轮数 >= min_turns (默认2)
  ├─ 条件2: 用户回答包含具体例子
  └─ 条件3: 不超过 max_turns (默认5)
  │
  ▼
generate_answer_card_analysis() 生成分析
  │
  ├─→ 调用 answer_card_summary.yaml 提示词
  ├─→ 传入对话历史
  └─→ LLM生成结构化分析
  │
  ▼
返回 AnswerCard:
  ├─ ai_summary: 核心观点概括
  ├─ ai_analysis: 深层分析
  ├─ key_insights: 关键洞察列表
  └─ user_answer: 用户回答摘要
```

## 九、架构演进方向

### 9.1 当前架构特点

1. **简单模式为主**：当前主要使用 `simple_chat.py`，直接LLM调用，流程简单可控
2. **LangGraph备用**：`chat.py` 和 `core/agent/` 提供复杂的ReAct智能体能力，但当前未启用
3. **文件存储为主**：对话记录、进度等使用文件系统存储，便于调试和导出
4. **双轨并行**：简单模式和完整模式API并存，可随时切换

### 9.2 未来演进建议

1. **A2A协议支持**：Agent-to-Agent通信，支持多Agent协作
2. **AG-UI协议**：标准化AI交互界面组件
3. **组件库封装**：统一前端组件，提高一致性
4. **智能体增强**：重新启用并优化LangGraph模式，提供更智能的引导
