# 方案润色后

### **1. 核心角色与总体任务**

- **角色**：职业规划咨询师，亲切、专业、善于引导。
- **目标**：通过系统化的筛选流程，帮助用户从之前探索的价值观、优势、热爱、工作目的中，最终确定三个职业发展方向。
- **交互原则**：
    - 每次只提一个问题，给用户充分的回答空间。
    - 当用户卡顿时，提供温和的引导，而非直接给答案。
    - 判断对话结束的条件：用户已选出 top 3 方向并确认。
    - 对话结束时调用 `end_conversation()` 函数生成`conclusion_card`。

---

### **2. 页面概览与设计**

页面主要分为两大部分。

1. **对话框区域（右侧）**：
    - **功能**：
        1. **流程引导（Cue）**：当进入一个新的交互模块时，AI根据预设的开场语和当前模块功能，发送引导性话语，开启该模块的对话。
        2. **细节对话**：基于当前表格内容和用户选中的表格行，AI与用户进行深入讨论，帮助用户厘清疑惑，辅助其做出选择。
2. **可编辑表格区域（左侧）**：
    - **功能**：作为核心数据载体和交互工具。用户在特定步骤中对表格进行编辑。
    - **UI/UX 规范**：
        - **滚动**：当表格内容过多时，提供水平和垂直滚动条。
        - **可编辑性**：
            - 每一步中，只允许用户编辑指定的列（如下拉选择或文本输入），其余列均为只读状态。
            - 用户点击“确认”按钮后，整个表格立即切换为**不可编辑**状态，直至后端返回新的表格数据，前端重新渲染后，再切换回**可编辑**状态。详情见figma原型图。
        - **按钮状态**：“Confirm”按钮在表格处于可编辑状态时启用；在表格处于不可编辑状态时置灰。
        - **行点击交互**：发生在每个流程用户点击“确认”按钮之前。用户可以点击表格中的任意一行，该行高亮显示，并与AI进行对话。**对话主题严格限定于当前步骤的主题和该行的内容**。

### **3. 交互流程详述**

### **第一步：开场白与准备**

- **前端 - 对话框**：【PROMPT - 开场白】
    
    ```
    你是一名专业的职业规划咨询师。现在，你正在与用户进行最后一轮咨询，目标是帮助他们确定三个最终的职业发展方向。请根据以下要求生成开场白：
    1. 亲切地问候用户，并祝贺他们进入了最后一轮，这是非常关键的一步。
    2. 清晰地说明本次对话的目标：我们将一起筛选和讨论，最终确定三个可以尝试的职业发展方向。
    3. 同时，温和地告知用户，如果对某些问题暂时没有答案，可以直接表达“不知道”或“不确定”，我们一步步来。
    4. 询问用户是否准备好开始。在用户给出肯定答复后，再进入下一步。
    5. 开场白应简洁、温暖、有力量感，避免过于机械。
    ```
    
- **前端 - 表格**：
    - **内容**：空表格。
    - **状态**：表格不可编辑。“Confirm”按钮置灰，鼠标悬浮时显示禁止图标。
    - 行不可选中
- **用户确认开始**：用户回复“准备好了”或类似意图。

### **第二步：步骤一 - 优势标记与筛选**

**主题**：展示热爱与优势的组合

**后端**

- 生成表格。
    - **输入**：用户的热爱列表（passion）和优势列表（strengths）【已过滤标记为“不确定”的优势】。
    - **函数**：`gen_table(strengths, passion)`
        - 生成表格，列：`id`、`热爱`、`优势`。
    - 返回表格给前端。

**前端**

- 对话框：【PROMPT - 步骤一引导】
    
    ```
    你是一名专业的职业规划咨询师。目前正处于“展示热爱与优势组合”环节。根据以下要求生成一段引导语：
    1. 向用户解释左侧表格的内容：这是将你的热爱列表和优势列表进行排列组合后的结果。优势列表已过滤为用户标记为“不确定”的优势。
    2. 如果用户对表格中的某一行有疑问，可以点击该行，然后在对话框中随时提出问题。但是表格内容不可更改。
    3. 引导用户浏览表格内容，并告知用户点击下方的“确认”按钮，我们将进入下一步的深入分析。
    3. 语气亲切、平和，让用户感到这是一个轻松的浏览确认环节。
    ```
    

**前端**

- 表格：
    - 内容：后端返回的表格结果。列：`id`、`热爱`、`优势`。
    - **编辑状态**：表格为**不可编辑**状态。所有单元格均为只读，“Confirm”按钮处于启用状态（因为无需编辑，用户可直接确认）。
    - 行可选中。

**用户交互**

- 用户可选中任意一行在对话框中提问交互。

**后端**

- 用户点击“确认”按钮后，进入下一步。

### **第三步：步骤二 - 匹配性分析与筛选**

- **后端**：`match_analysis(table_step2)`
    - **输入**：`table_step2`。
    - **处理**：为每一行分析“热爱”与“优势”的匹配性。
    - **输出**：表格 `table_step2_with_analysis`，列：`id`, `热爱`, `优势`, `匹配性`。
- **前端 - 对话框**：【PROMPT - 步骤二引导】
    
    ```
    你是一名专业的职业规划咨询师。目前正处于“匹配性分析”环节。根据以下要求生成一段引导语：
    1. 向用户解释左侧表格的新增内容：我们对所有的组合进行了分析，判断“热爱”和“优势”是否匹配。
    2. 引导用户审阅这些分析结果。如果用户认同分析，可以保持原样；如果不认同，可以在表格的“匹配性”列手动修改为“匹配”。
    3. 如果用户对表格中的某一行有疑问，可以点击该行，然后在对话框中随时提出问题。
    4. 告诉用户，确认后点击“确认”按钮，进入下一步。
    ```
    
- **前端 - 表格**：
    - **内容**：`table_step2_with_analysis`。
    - **状态**：可编辑。可编辑列：`匹配性`（下拉选项：匹配、不匹配）。
    - 行可选中。
- **用户交互**：用户在表格中确认或修改，点击“确认”按钮。
    - **前端动作**：将用户编辑后的 `table_step2_edited` 发送给后端。表格切换为不可编辑，“Confirm”按钮置灰。
    - 行不可选中。
- **后端**：`filter_match(table_step2_edited)`
    - **处理**：删除`匹配性`为“不匹配”的行，并删除`不匹配原因`列。
    - **输出**：过滤后的表格 `table_step3`，列：`id`, `热爱`, `优势`。

```

    - **内容**：`table_step3_with_hypo`。

### **第八步：最终选择与报告解锁**

- **前端 - 对话框**：【PROMPT - 最终选择引导】
    
    ```
    你是一名专业的职业规划咨询师。现在，我们已经完成了所有的筛选步骤，来到了最终的决策时刻。根据以下要求生成一段引导语：

    2. 引导用户从这些方向中，选出三件最让他们心动、最想尝试去做的事。
    3. 采用引导式提问，每次只问一个问题，例如：“在这几个方向中，哪个最让你有立刻行动的冲动？”、“如果只能选一个作为你的第一优先，会是哪个？为什么？”
