## 项目功能总览（BeingDoing「四维共鸣」）

### 一、核心业务流程

- **激活码驱动的探索流程**
  - 用户拿到激活码，在首页或 `/explore/activate` 输入并验证。
  - 后端用 `simple-auth` 创建 / 激活 `activation_code`，映射到唯一 `session_id`，所有 simple-chat 数据以此为根。
  - 验证成功后，根据是否已完成问卷，跳转到：
    - `/explore/survey`：未完成问卷 → 填写基础信息；
    - `/explore/chat/[phase]`：已完成问卷 → 直接进入当前阶段的 AI 探索。

- **四维探索（values / strengths / interests / purpose）**
  - 每个维度对应一个独立的 Chat 页面 `/explore/chat/[phase]`：
    - 维度：信念（values）、禀赋（strengths）、热忱（interests_goals）、使命（purpose）。
    - 与后端 `simple-chat` 流程对接：
      - `POST /api/v1/simple-chat/init`：初始化阶段，无历史则生成首轮问题；
      - `GET /api/v1/simple-chat/history`：获取历史消息；
      - `POST /api/v1/simple-chat/message/stream`：流式发送消息。
  - 前端本地维护 `ExploreSession`（`src/frontend/lib/explore/session.ts`）：
    - `activationCode`、`currentPhase`、`unlockedPhases`、`surveyCompleted` 等；
    - 阶段完成后调用 `unlockNextPhase()` 解锁下一个维度。

- **多线程对话（同一维度最多 5 个线程）**
  - 每个维度下可创建最多 5 个独立的对话线程（thread）：
    - 本地存储：`src/frontend/lib/explore/threads.ts`，以 `activation_code + phase` 为 key 存储；
    - 每个 `ChatThread` 含：`id`、`title`、`status`（in-progress/completed）、`messages`、`createdAt`。
  - `/explore/chat/[phase]` 左侧为线程列表：
    - 新建对话：最多 5 个，使用 `simple-chat/init` 且附带 `thread_id`，后端按 `{phase}__{thread_id}` 独立存储；
    - 删除对话：从本地 threads 中移除，若删除当前 thread，会自动切到下一个 / 新建一个。
  - 线程状态：
    - 用户确认当前对话完成 → 线程标记为 `completed`；
    - 只有选中一个 `completed` 的线程时，顶部「完成并继续」按钮才可用。

- **阶段间过渡 & 报告预备**
  - 完成某一阶段后：
    - 点击「完成并继续」 → 跳转 `/explore/transition?from={phase}` 显示过渡页；
    - 过渡页提示：已完成的维度、当前进度（进度条）、下一步要做的维度、休息提示。
  - 完成所有四维后：
    - 从最后一个阶段（purpose）进入 `/explore/report` 报告预备页；
    - 报告生成逻辑目前以占位为主，详细协议在 `uidesign/reference/dashboard-api.md` 中。

- **报告与历史**
  - `/explore/report`：综合报告预备页（当前主要为引导文案和「即将上线」提示）。
  - `/explore/report/view`：报告展示页（占位，后续挂接真实报告生成接口）。
  - `/explore/history`：历史探索记录/报告入口（占位页，未来可展示历史 activation 与报告列表）。

### 二、前端页面结构清单（Next App Router）

#### 1. 根布局与主 Layout

- `src/frontend/app/layout.tsx`
  - 顶级 HTML 布局，设置 `data-theme="ideal"`、`data-color-scheme="light"`，注入字体变量。
  - 注入一段 inline script：从 `localStorage.bd-theme` 恢复主题配色与 color-scheme。

- `src/frontend/app/(main)/layout.tsx`（`MainLayout`）
  - 应用于 `(main)` 分组中的所有页面：
    - 背景：`bd-aurora-bg` + `bd-eff-bg` + `bg-bd-bg`；
    - 顶部导航：`TopNavbar`（动态加载，包含品牌、主导航、语言切换、登录/头像、admin 入口）；
    - 主体：`<main className="pt-14">` 包裹 `AuthGate`，未登录用户可触发登录弹窗。

#### 2. 首页与静态介绍页

- `/` → `src/frontend/app/(main)/page.tsx`（`LandingPage`）
  - Hero 区：品牌 Slogan、入口 CTA（开始探索 / 查看理论 / 社区）。
  - 「四维探索卡片」区：四张卡片介绍信念/禀赋/热忱/使命。
  - 连接区：四条弧线 + 垂直白色光柱 → 汇聚到下方报告卡片。
  - 报告 CTA 卡片：`bd-report-glass-card`，标题「获取『四维共鸣』专属报告」，按钮跳转 `/explore/intro`。
  - 「即将上线」区：职业双轨 / 光谱共振 / 静室之我 卡片。
  - 用户故事 / Testimonials：多列故事卡片，带光影效果。

- `/theory` → `TheoryPage`
  - 介绍理论背景（四维模型、职业发展理论等），静态内容。

- `/about` → `AboutPage`
  - 团队介绍、产品愿景、项目背景等。

- `/community` → `CommunityPage`
  - 社区相关介绍（目前为占位/静态文案，未来可挂外部社区链接）。

#### 3. 探索入口与激活

- `/explore` → `ExploreChoicePage`
  - 探索入口选择页：
    - 若尚未激活：提示先输入激活码；
    - 若已有 session：提供「继续探索」/「查看报告」等入口。

- `/explore/intro` → `ExploreIntroPage`
  - 探索引导页，类似 Onboarding：
    - 顶部：简要说明探索目的。
    - 中部：卡片形式介绍四个维度（使用 i18n 文案）。
    - 底部 CTA：开始探索 / 使用激活码。

- `/explore/activate` → `ActivatePage`
  - 输入激活码页：
    - 组件逻辑：
      - 输入 `code`，调用 `POST /api/v1/simple-auth/activate`；
      - 成功后保存 `activationCode` 与 `sessionId` 到 `ExploreSession`（`saveSession`）；
      - 根据 `surveyCompleted` 判定跳转 `/explore/survey` 或 `/explore/chat/{currentPhase}`。
    - 已有激活码时，从 `getLastActivationCode()` 自动填充。

#### 4. 问卷与阶段间摘要

- `/explore/survey` → `SurveyPage`
  - 探索前基础问卷：
    - 获取/保存接口：`surveyApi.getForActivation` / `surveyApi.saveForActivation`；
    - 使用 `activation_code` 作为 key，数据存于 `data/simple/{session_id}/basic_info.json`。
  - 支持「跳过」与「保存并继续」，会更新 `ExploreSession.surveyCompleted`。

- 阶段间摘要组件（嵌在某些页面/设置中）：
  - `src/frontend/components/survey/PriorContextPanel.tsx`
    - 用于查看/编辑阶段间摘要（例如 values → strengths 的过渡）。
    - API：`/api/v1/simple-chat/prior-context` GET/POST。

#### 5. 四维对话页面

- `/explore/chat/[phase]` → `ChatPhasePage`
  - 动态路由，`phase` 为 `values | strengths | interests | purpose`：
    - 导航保护：若未激活或 phase 未解锁 → 重定向到 `/explore/activate` 或当前解锁阶段；
    - 使用 `ExploreSession` 确定当前阶段、已解锁阶段。

  - 左侧：线程侧边栏 `ChatPhaseSidebar`
    - 显示当前维度下的所有线程（最多 5 个）：
      - title、最后一句预览、最后消息时间；
      - 状态 Tag：进行中（橙色）、已完成（绿色）。
    - 支持：
      - 新建对话：`+ 新建对话` → 调用 `simple-chat/init`（带 thread_id），新增本地 `ChatThread` 并切到该线程；
      - 选择线程：点击切换当前激活线程；
      - 删除线程：垃圾桶按钮 + 确认弹窗，调用本地 `removeThread`，如删的是当前线程，会自动切换/新建。

  - 右侧顶部：阶段标题 + 完成并继续
    - 显示当前阶段名及简短描述。
    - 「完成并继续」按钮：
      - 只有当当前选中线程的 `status === 'completed'` 时可用；
      - 点击后：
        - 调用 `unlockNextPhase` 更新 `ExploreSession`；
        - 跳转 `/explore/transition?from={phase}` 或最终 `/explore/report`。

  - 对话展示框：
    - 上方固定标题：「正在探索 · 信念/禀赋/热忱/使命 维度」；
    - 消息流：
      - 用户消息：右侧白色气泡，支持复制；
      - AI 消息：左侧无气泡，仅左侧色条，`FlowAiMessage` 渲染 Markdown 内容，支持复制/重新生成；
      - 每条消息带时间戳（HH:MM）。
    - 内部滚动，右下角有「向下箭头」快速滚动到底部。

  - 输入区：
    - 底部固定，textarea 自适应高度（最多约 7 行），Enter 发送，Shift+Enter 换行；
    - 发送按钮在发送中会变为停止按钮，支持中断流式。

  - 「确认完成」按钮：
    - 位于输入区下方；
    - 用户认为当前线程已得到满意结论时点击 → 将线程状态标记为 `completed`，并在侧边栏以绿色标签展示。

#### 6. 阶段过渡与报告

- `/explore/transition` → `ExploreTransitionPage`
  - 显示从某一阶段完成到下一阶段开启之间的过渡：
    - 参数：`from=values|strengths|interests|purpose`；
    - 内容：
      - 当前已完成阶段名；
      - 整体探索进度条（X / 4 已完成）；
      - 下一步要进入的阶段说明；
      - CTA 按钮：继续探索 / 查看报告。

- `/explore/report` → `ReportPrepPage`
  - 报告预备页，提示即将生成四维共鸣报告，当前为占位逻辑。

- `/explore/report/view` → `ReportViewPage`
  - 报告详情页，占位（未来挂接真实报告生成与展示）。

- `/explore/history` → `HistoryPage`
  - 历史探索记录页，占位（未来可展示历史 activation / 报告列表）。

#### 7. 个人空间 / Dashboard

所有 Dashboard 路由共用 `DashboardLayout`：

- `src/frontend/app/(main)/dashboard/layout.tsx`

各页面功能（当前部分为占位骨架，未来可依据 `dashboard-api.md` 接入）：

#### 8. Admin 管理控制台（仅 super_admin）

统一在 `/admin` 下，使用 `AdminLayout`：

- `src/frontend/app/(main)/admin/layout.tsx`
  - 左侧 Admin 侧边栏：
    - Dashboard（`/admin`）；
    - 激活码管理（`/admin/activations`）；
    - 对话与阶段（`/admin/conversations`）；
    - 报告概览（`/admin/reports`）；

    - 日志与调试（`/admin/logs`）；
    - 系统设置（`/admin/system`）。
  - 权限保护：
    - 通过 `authStore.user.is_super_admin` 控制；
    - 非 super_admin 访问 `/admin/*` 时，显示无权限提示。

具体子页面：

- `/admin` → 管理总览（占位 Dashboard）
  - 4 个关键指标占位卡片；
  - 完成率 / 趋势图占位区域。

- `/admin/activations` → 激活码管理（已联通后端）
  - 调用 `GET /api/v1/admin/activations`，从 `data/simple/activations.json` 中读出所有激活码记录；
  - 支持过滤：
    - 状态：全部 / active / expired / revoked；
    - 模式：全部 / values / strengths / interests / combined；
    - 搜索：按 activation_code / session_id 模糊匹配；
  - 表格列：
    - 激活码、Session ID、模式、状态、创建时间、过期时间、最后活跃时间。

- `/admin/conversations` → 对话与阶段（占位）
  - 说明将来会使用：
    - `/api/v1/admin/analytics/chat-records`；
    - `/api/v1/admin/analytics/session-detail`。

- `/admin/reports` → 报告概览（占位）
  - 用于按 activation_code / 用户查看报告生成情况，未来可接入报告列表与预览。

- `/admin/analytics` → 埋点与 Token 统计（占位）
  - 将来展示 simple-chat 与报告生成的 Token / 调用量 / 错误率等。

- `/admin/logs` → 日志与调试（占位）
  - 用于查看 runs.jsonl 中的记录、被点赞的消息等：
    - `/api/v1/admin/analytics/like-detail`。

- `/admin/system` → 系统设置（只读占位）
  - 展示 APP_ENV / ARCHITECTURE_MODE / LLM_PROVIDER 等环境配置，避免高危写操作。

### 三、后端核心模块与接口（简要）

- 激活码管理：`src/backend/app/utils/simple_activation_manager.py`
  - 文件存储：`data/simple/activations.json`；
  - 创建 / 获取 / 更新最后活跃时间 / 列出所有激活码。

- Simple 模式认证与对话：`src/backend/app/api/v1/simple_auth.py`、`simple_chat.py`
  - `/simple-auth/activation`：创建激活码；
  - `/simple-auth/activate`：通过激活码获取 session 信息；
  - `/simple-chat/init`：阶段初始化；
  - `/simple-chat/history`：阶段历史；
  - `/simple-chat/message` & `/message/stream`：对话消息；
  - `/simple-chat/survey`：问卷；
  - `/simple-chat/prior-context`：阶段间摘要。

- Admin API：`src/backend/app/api/v1/admin.py`
  - `/admin/analytics`：仪表盘统计；
  - `/admin/analytics/chat-records` / `session-detail` / `like-detail`：对话和日志详情；
  - `/admin/activations`：本次新增，用于 Admin 激活码列表。

### 四、未复原的「之前版本」功能（说明）

根据当前代码结构和 `dashboard-api.md` 文档，之前你记得的某些功能（例如：

- 激活码的**批量创建 / 删除 / 标记过期**（前端可视化操作）；
- 报告的**查看 / 删除 / 导出**；
- 更完善的**对话明细表格与详情**页面；

在当前 repo 中主要保留了**后端基础能力和文档设计**，前端实际的 Admin/Dashboard UI 部分只实现了「骨架 + 部分联通」（比如激活码列表）。  

这份清单可以作为你对比「现在版本 vs 记忆中的旧版本」的基础，我们可以在这个结构上，逐个把你坚持要有的功能重新实装回来（例如下一步就可以在 `/admin/activations` 上补「新建激活码、标记过期」这类按钮与写操作）。
