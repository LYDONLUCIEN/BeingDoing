本文档汇总近期围绕「沉淀七步筛选表、终步提交、进度与导航」的一整批改动，便于复盘、提交说明与后续跟进。

## 一、改动总览

### 1. 后端（`src/backend/app/api/v1/simple_chat_routes.py`）

| 主题 | 说明 |
|------|------|
| Step 6 保险快照 | 短链跳过第 6 步 UI 时，通过 `_rumination_persist_skip_to_step7(..., preserve_step6_initial=...)` 仍写入 `snapshots["6"]["initial"]`（无 `submitted`），便于「上一阶段」回看表格形态（能看即可）。 |
| 第 7 步提交与过渡 | 第 7 步提交成功后使用 `rumination_finalize_transition`（`next_action`），不再走结论卡插入路径；校验文案等处有小修正（如「勾选」→「点选」）。 |

### 2. 前端 — 聊天页（`src/frontend/app/(main)/explore/chat/[phase]/page.tsx`）

| 主题 | 说明 |
|------|------|
| 第 7 步终局弹窗 | 表格「确认」在 step 7 时先打开确认弹窗（确认后不可再改 → 再 `submitTable`）；成功且 `next_action === rumination_finalize_transition` 后 `thread/complete`、解锁阶段、`router.push('/explore/transition?from=rumination')`。 |
| 闪屏优化 | 终局成功路径避免先 `setRuminationProgressState` 等触发「final_choice 约 82%」一帧再跳转；减少跳转前多余 state 更新。 |
| 顶栏「完成并继续」 | 沉淀阶段在筛选第 7 子步（`ruminationViewStep === 7`）或 `final_choice` 未完结等场景下隐藏。 |
| 子步边界 | `loadRuminationTableStep` / 初始化拉表时对 step 边界写入改为同步使用当时 `messagesRef` 长度，减少 `queueMicrotask` 导致边界晚于首条消息、消息被切片裁掉的问题。 |
| 进度条导航 | 接入 `isRuminationFilterStepReachable`：`loadRuminationTableStep` 非管理员时对不可达步直接 return；上一阶段/下一阶段与进度条分段一致。 |
| 移除终步提示气泡 | 去掉第 7 步下方 `finalTableTextConfirmHint` 的 `FlowAiMessage`。 |

### 3. 前端 — 进度与导航（`src/frontend/lib/explore/ruminationProgressNav.ts`）

| 主题 | 说明 |
|------|------|
| `isRuminationFilterStepReachable` | **已提交**（`submitted`）的步可回看；未提交步仅当 `step === filter_step` 可进入。**不把仅有 `initial` 的快照当作已到达**（修复短链保险快照误开 step6、与 step5 不同步的观感）。 |
| `computeFurthestBrowsableFilterStep` | 去掉对「initial-only」的放大；文档注明进度条勿再用单一 `furthest` 做 `step<=N` 判断。 |

### 4. 前端 — 进度条 UI（`src/frontend/components/explore/RuminationSectionProgress.tsx`）

- 七段可点区域按 `isRuminationFilterStepReachable(step, displayProgress)` 禁用/启用，**移除对 `segmentJump.furthestStep` 的依赖**（父组件不再传 `furthestStep`）。

### 5. 前端 — 筛选表组件（`src/frontend/components/explore/RuminationTableWidget.tsx`）

- `cellDisabled = disabled || confirmDisabledAfterCommit`：本步已确认后，**整表只读**（含行选中、多选、单元格、假设单选/再生、确认）；**「重新填写」** 仍仅受页面级 `disabled`（发送中等）限制。

### 6. 前端 — 消息切片（`src/frontend/lib/explore/ruminationStepBoundaries.ts`）

- 查看当前筛选子步且边界异常（起点等于 `messages.length`）时，回退用上一子步边界，避免本步首条消息被整段裁空（与同步边界改动配合）。

### 7. 前端 — 进度展示（`src/frontend/components/explore/RuminationSectionProgress.tsx` 内里程碑逻辑）

- `computeDisplayedRuminationMilestone`：`filter_step >= 7` 时对「已提交档位数」做 `effectiveSubmitted` 下限处理，缓解短链未每步写 `submitted` 时进度条仍显示约 50% 的问题。

### 8. 国际化（`src/frontend/lib/i18n/locales/zh.ts`、`en.ts`）

- 新增第 7 步终局弹窗文案：`step7FinalizeTitle`、`step7FinalizeMessage`；按钮复用 `common.cancel` / `common.confirm`。

### 9. API 类型/注释（`src/frontend/lib/api/rumination.ts`）

- `next_action` 补充说明含 `rumination_finalize_transition`。

## 二、问题 → 处理对照（便于写 CHANGELOG）

1. **短链跳 7 后「上一阶段」看不到 step6** → 后端写入 step6 `initial` 快照。  
2. **重置后再走完 7/7 进度条仍偏低** → 里程碑展示对 `filter_step >= 7` 与 submitted 计数对齐。  
3. **终步不要结论卡、要阻断弹窗再进过渡页** → 弹窗 + `rumination_finalize_transition` + 跳转过渡页；隐藏终步多余提示与顶栏完成按钮（按子步）。  
4. **终局跳转前闪 ~82%** → 避免在跳转前把 progress 推到 `final_choice` 展示态。  
5. **第 7 步首条消息消失** → 边界同步写入 + 切片兜底。  
6. **step6 能点进、step5 像被跳过** → 可达性只认 `submitted` 与当前 `filter_step`，不认纯 `initial`。  
7. **确认后表格仍可点行/改格** → `RuminationTableWidget` 全面 `cellDisabled`。
