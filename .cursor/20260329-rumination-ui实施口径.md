# Rumination UI 实施口径（2026-03-29）

> 样式参考：`uidesign/beautiful/rumination-finall.html`（仅视觉与交互参考，**不迁移其演示逻辑**）。  
> 业务与 API：**沿用现有** `simple_chat`、`rumination_*` 等后端逻辑，本文件只约束**前端布局、展示分层、文案与断点**。

---

## 一、范围边界

| 项目 | 口径 |
|------|------|
| **布局** | **仅 `rumination`（沉淀）阶段**使用左右分栏；**values / strengths / interests / purpose 保持现有页面不变**。 |
| **左右比例** | 左侧表格卡片与右侧对话区域 **约 1:1（50% : 50%）**，同一主内容高度内 **等高对齐**，视觉平衡。 |
| **后端** | **不改为多会话拆 thread**；仍使用当前激活码下的 rumination 线程与 `message/stream` 等接口。 |
| **Dev 面板** | **不实现**（设计稿左上角调参面板不上线）。 |

---

## 二、左侧：表格卡片（样式与行为）

- 复刻设计稿中的 **毛玻璃卡片**（背景、圆角、边框、阴影）、**表头 sticky**、**行 hover / 选中高亮**（薰衣草主题：`#E8DFFE`、左侧竖条、选中行内下拉样式）。
- **列宽**：按 **当前步骤真实列** 动态分配，**不死抄** HTML 静态比例。
- **溢出**：单屏宽度不足时，表格区域 **横向滚动**（`overflow-x: auto`），避免挤压右栏。
- **下拉 / 其他选项**：与现有 `RuminationTableWidget` 行为对齐；样式对齐设计稿（圆角、自定义箭头）；「其他」展开输入需 **阻止冒泡** 避免误触行选中。
- **确认后 Loading**：使用设计稿中的 **弱化 blur** 遮罩（建议初始 `backdrop-filter` **20–32px**，见下文「性能与观感」）；文案走 i18n。

---

## 三、右侧：历史时间线 + 按 Step 分段展示

### 3.1 用户可见原则

- 右侧展示 **同一条后端会话** 的 **完整历史时间线**。
- 历史需 **按「表格筛选 Step」分段呈现**（与 `rumination_progress.filter_step` 或产品定义的 step 索引一致，见实现时对齐字段名）。

### 3.2 「新 Step 空白画布」规则（核心）

- 当用户 **已确认当前表格 Step**，进入 **下一个 Step** 时：
  - **后端**：继续在同一线程追加消息，**不删历史、不换 session**。
  - **前端**：在 **当前 Step 的视图层** 表现为 **全新空白**（仅显示该 Step 起始后的新对话；或等价实现：**只渲染属于当前 Step 的消息**）。
- 当用户 **回到上一个 Step**（若产品允许回看/编辑策略）时：
  - **前端**：再次 **显示该 Step 下曾出现过的聊天记录**（与当时一致）。

### 3.3 实现层建议（供开发拆任务）

**已落地（方案 B）**：`src/frontend/lib/explore/ruminationStepBoundaries.ts` — 按 `activationCode + threadId` 将 **筛选子步 → 消息数组起始下标** 存 `localStorage`；右侧列表为 `messages.slice(start, nextStepStart)` 并排除 `table_widget`。进入新子步时在 **表格提交成功切步** 与 **首次加载某子步表格** 时写入下标。  

**技术债**：无后端 `table_step` 元数据时，**换设备/清缓存** 后子步与气泡对齐可能漂移；**重新生成** 等会截断消息列表的操作可能使下标失真 —— 后续可迁 **方案 A**（消息带 `rumination_filter_step`）。

### 3.4 气泡与思考区

- **复用现有气泡组件**（如 `FlowAiMessage`）。
- **保留**「有思考则显示思考区」的 **逻辑**；仅做 **换皮**（颜色、圆角、阴影与右栏容器一致）。
- **系统内嵌提示**（如「对话已完成」等）—— **不改文案逻辑与触发条件**，仅随全局样式必要时微调间距。

## 四、输入框：共用组件样式增强

- 在 **不破坏其他阶段** 的前提下，为 rumination 布局 **传入变体 / className**（或专用包装组件），复刻设计稿：
  - 圆角条、默认与 **选中表格行** 时的 **高亮边框 + 外圈浅紫 glow**（`focused-context`）。
  - 选中行时左侧 **inline 图标**；placeholder 可带行上下文提示（i18n）。
- **发送按钮**：空内容禁用态 / 有内容激活态，样式对齐设计稿。

## 五、响应式：极限宽度与上下堆叠

- **已定稿**：监听沉淀 **工作台容器**（`.rumination-workbench`）实际宽度 `w`，与视口宽度 `vw` 比较；当 **`w < (2/3) × vw`** 时切换为 **上下堆叠**（**上表下聊**）；否则 **左右 50:50 并排**（`flex-row` + 两栏 `flex-1`）。
- 使用 `ResizeObserver` + `window.resize`，避免纯 CSS `vw` 媒体查询与「侧栏占宽」不一致的问题。
- 目的：覆盖 **窄浏览器窗口、分屏、高缩放**，**非**单独做移动端 App。

## 六、国际化（中英文各一版）

- 本页涉及的所有 **可见文案**（标题、表头占位、按钮、placeholder、loading 提示、空状态等）均通过 **i18n** 提供 **中文 `zh`、英文 `en`** 两套词条。
- 不在组件内硬编码单一语言字符串。
- 建议 key 前缀：`explore.rumination.workbench.*`（或项目现有规范）。

## 七、性能与观感：Loading Blur

| 档位 | 大致效果 |
|------|----------|
| **减弱**（推荐初值 20–32px） | 背后表格仍隐约可见，GPU 压力小，弱设备更稳。 |
| **强**（如 64–80px） | 背景几乎糊满，更「磨砂玻璃」，高级感略强，低配机可能掉帧。 |

**初版采用减弱**；若验收希望更强质感，再单独调参（仅 CSS 变量）。

## 八、实施顺序建议

1. 新增 rumination **专用布局壳**（路由仍为 `/explore/chat/rumination` 或同级，**仅在该 phase 渲染**）。  
2. 抽 **CSS 变量 / 模块样式**（薰衣草卡片、选中行、右栏气泡容器）。  
3. 左栏接入 **现有表格数据与 submit**，加 **横向滚动** 与 **选中行 state**。  
4. 右栏接入 **现有消息列表 + stream**，**换皮**；实现 **按 Step 过滤/分段**（优先推动消息元数据方案）。  
5. **输入框变体** + 选中行联动。  
6. **断点** 上下堆叠。  
7. **中英词条** 全覆盖。  
8. 与设计稿 **像素级走查**（间距、圆角、动画时长）。

## 九、风险与依赖

| 风险 | 说明 |
|------|------|
| **Step 与消息绑定** | 若无后端元数据，仅靠前端索引，**跨设备不一致**；应尽早定方案 A。 |
| **「回到上一 Step」** | 若产品允许修改历史 Step，需明确是否 **只读回看** 还是 **可继续发消息**（影响是否解锁该 Step 的输入）。 |
| **双栏 + 侧栏** | 现有 `ChatPhaseSidebar` 与顶栏并存时，需预留宽度，避免 50% 实际过窄。 |
